Sorry, We Weren't Able to Process Your OGC Service

Here is a list of the most common errors users face:

  • OGC Compliance: This system only supports OGC (Open Geospatial Consortium) WMS and WFS services. If this service has been uploaded through NGDS from an ESRI Shapefile or a CSV file, then this is an OGC service served through Geoserver. If this is a linked data service, it is possible that it does not conform to OGC specifications.
    • Confusion: ArcGIS Server distinguishes between an ArcGIS REST Service and an OGC Service. A service endpoint for an ArcGIS REST Service might look something like this:

      http://services.azgs.az.gov/arcgis/rest/services/aasggeothermal/TXActiveFaults/MapServer

      Whereas a service endpoint for an ArcGIS OGC WFS Service might look something like this:

      http://services.azgs.az.gov/arcgis/services/aasggeothermal/TXActiveFaults/MapServer/WFSServer?
