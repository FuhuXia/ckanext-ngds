Help

Here you will find additional information that can help you successfully use the following site features to access data through the National Geothermal Data System (NGDS):

  • Search feature
  • Map interface
  • Library

Search Feature

This site’s search feature provides access to a catalog of available geothermal resource data by indexing a database of metadata about those resources. 

Metadata is basically “data about the data” – it is keywords and terms that the data contributors have used to describe each data resource.  When you type a search word or phrase into the search box, the NGDS compares it to the metadata records and returns the most appropriate records.  Information on how to access a given data resource is contained in the metadata records returned in the search results.

Map Interface

The map interface is typically used when you want to see a geographic distribution of data sites that may be of interest to you.

You can use the map to search the NGDS either by typing a keyword into the search box or by using your mouse to draw a “bounding box.” A bounding box delineates geographic parameters on the map and the system retrieves the records that fall within those parameters.

Search results are displayed in list form and as colored circles on the map.

Map controls are as follows:

  • Search by Keyword (): Enter search terms and press the Enter key; the results will appear in list form beneath the search field as well as on the map.
  • Zoom In (): Zooms in on the map.
  • Zoom Out (): Zooms out.
  • Draw a Rectangle (or "Bounding Box") (): Use this tool to search for results within a given area by drawing a rectangle on the map. Any record describing features within the bounding box you drew will appear as search results.
  • Zoom to Extent (): Zoom to the original extent of the map.
  • Active Layer Menu (): Mouseover for a toggle-able list of active layers in the map. This menu is populated as users choose to add available WMS resources to the map from their search queries.
  • Search Results Menu (): Toggle a menu of search results. This widget has some automatic capabilities; if a user makes a query by bounding box or keyword, the search results menu will automatically toggle itself.

Library

The Library interface allows you to use keywords and phrases to search the data records. Once you have results, you can narrow them to find the records you are looking for by choosing a set of criteria by which to filter your results.

Narrowing Your Search Results

  1. Type a keyword or phrase into the search box to receive an initial set of results.
  2. Choose one or more of the filter topics from the left side of the screen:
  3. Within each topic, check the box by the criteria you wish to use as your filter(s):

    Selecting a check box will automatically update your search results to show only data that match the criteria you have selected. Note that you can use multiple criteria simultaneously to filter your results.
  4. To remove a filter, select the name of the filter at the top of the screen:

You can filter your search results using the following topics:

  • Category: Filter results by category and subcategory. For example, the category “drilling technology” includes a variety of subcategories related to drilling, such as the presence or absence of casing diagrams.
  • Format: Filter results by format of the documents or datasets. Formats include data types and online resources, such as Web services.
  • Source & Author: Filter results by data source and/or author.
  • Status: Filter results by the status of the data within the context of the NGDS project. Data status tends to reflect the state of completion of that data. For example, typical NGDS documents or datasets will be either completed, ongoing, or deprecated.
  • Content Model: Filter results by the content model to which the data conforms. All Tier 3 NGDS data conforms to a specific content model, which reflect properties of a given data set. A search for a specific content model would return results of data structured to conform to that content model.
  • Public/Private: Filter results by the visibility of the data. When published, data contributors can designate whether or not a given document or dataset is publicly accessible or private. A setting of “private” renders the dataset invisible in public searches.

A number of resources are available to NGDS users in the form of applications developed for NGDS and applications compatible with NGDS data. A full list of these are available on the Resources page.
