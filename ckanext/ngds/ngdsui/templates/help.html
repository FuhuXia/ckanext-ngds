General

The National Geothermal Data System (NGDS) is a distributed data system providing access to information resources related to geothermal energy from a network of data providers. Data are contributed by academic researchers, private industry, and state and federal agencies. Built on a scalable and open platform through the U.S. Geoscience Information Network (USGIN), NGDS respects data provenance while promoting shared resources. Since NGDS is built using a set of open protocols and standards, relying on the Open Geospatial Consortium (OGC) and International Organization for Standardization (ISO), members of the community may access the data in a variety of proprietary and open-source applications and software. In addition, developers can add functionality to the system by creating new applications based on the open protocols and standards of the NGDS.

The NGDS, supported by the U.S. Department of Energy’s Geothermal Technology Program, is intended to provide access to all types of geothermal data to enable geothermal analysis and widespread public use in an effort to reduce the risk of geothermal energy development.

This website, the NGDS Node-in-a-Box, is a CKAN extension designed to facilitate user access to NGDS data resources by providing a search interface for the NGDS metadata catalog. The NGDS metadata catalog is a collection of metadata records that describe NGDS data resources; users can use these metadata records to locate and access the data resources described by these metadata records. In addition, the NGDS Node-in-a-Box provides users with utilities that can be used to create and manage metadata records within the NGDS metadata catalog.

Searching

The NGDS metadata catalog performs searches based on metadata.

Metadata is data used for organizational purposes. Often, metadata is used to organize other types of data or data containers (the distinction between data and data containers is often inconsequential).

A public library is a classic example of metadata in action. In a public library, data is stored in books (which act as data containers, but are physically difficult to distinguish from the data they contain). Librarians use metadata to organize these books and the data they contain. Metadata describing books includes the book’s title, author, date of publication, and subject. A card catalog is a repository for metadata about library books.

In the case of the NGDS metadata catalog accessible through this website, metadata records perform the same functions as cards in a public library catalog: each metadata record in the catalog describes an NGDS data resource in ways that help users locate and access that resource.

Map

The map is used to search the NGDS metadata catalog; search results are displayed in list form in the search interface on the left side of the map; and as colored rectangles on the map representing the physical extent of the features described by each dataset. Map controls are as follows:

  • Search by Keyword (image): Search the metadata catalog by keyword(s); enter search terms and press the Enter key or click the magnifying glass button; the results will appear in list form beneath the search field as well as on the map
  • Draw a Rectangle (image): Use this tool to search for results within a given area by drawing a rectangle on the map. When you have drawn a rectangle, click the magnifying glass button; any dataset describing features within the box you have just drawn will appear as search results
  • Clear Rectangle (image): Use this tool to stop drawing a rectangle or remove a rectangle that you have drawn with the Draw a Rectangle tool
  • Clear Results (image): Click this button to remove all search results from the map
  • Show/Hide Results (image): Click this button to toggle the visibility of search results
  • Center Results (image): Click this button beneath a given search result to center those results on the map
  • Zoom In (image): Click this button to zoom in on the map
  • Zoom Out (image): Click this button to zoom out
  • Full Screen (image): Click this button to view the view the metadata catalog search in fullscreen mode; note that leaving fullscreen mode will clear your search results
  • Basemap (image): Mouse over this button to reveal a dropdown list of different basemaps. Basemaps are maps that are used as a background for the NGDS metadata catalog search map. Click the desired basemap in the basemap menu to change the basemap for the catalog search map. Basemaps can be changed at any time

Library

The Library page acts as a keyword search interface for the NGDS metadata catalog. Here, users can search through NGDS datasets by keyword and filter the results based on criteria that expand as more metadata records are added to the catalog. Search results can be filtered by multiple criteria simultaneously. Filter criteria are as follows:

  • Category: Filter results by category and subcategory. For example, one category is drilling technology, which includes a variety of subcategories related to drilling, such as the presence or absence of casing diagrams; clicking the checkbox associated with any these subcategories will exclude any datasets that do not include the indicated information.
  • Format: Filter results by dataset format; formats include data types and online resources, such as web services. Clicking the checkbox associated with any of these formats will exclude any datasets that do not match the indicated format(s).
  • Source & Author: Filter results by source and-or author; clicking the checkbox associated with a specific source or author will exclude any datasets not submitted by the indicated source(s) or author(
  • Status: Filter results by the status of the dataset within the context of the NGDS project; dataset status tends to reflect the state of completion of the dataset. Clicking the checkbox associated within a given state will exclude any datasets that fall outside the selected state.
  • Content Model: Filter results by the content model to which the dataset conforms. Each NGDS dataset conforms to a specific content model; clicking the checkbox beside a specific content model will exclude any datasets that do not conform to the indicated content model(s).
  • Public/Private: Filter results by the visibility of the dataset. When published, users can designate whether or not a given dataset is publicly accessible of listed privately; clicking the appropriate checkbox will exclude datasets on the basis of whether or not they are public or private.

Contribute

The Contribute page allows users to contribute to NGDS by creating and managing metadata records. To contribute, you need to sign up for an account and click Contribute; you will be prompted to log in to your account if you have not already logged in.

Having clicked Contribute and logged in, you will see the Contribute page. Here, you are presented with two options:

  • Individual Upload: Users create a collection and then add one or more metadata records to the collection; each metadata record describes a data resource. During the metadata record creation process, the user can bind the metadata record to the dataset it describes in one of two ways:
    • By providing a hyperlink to a preexisting web-accessible dataset
    • By uploading their data to the Node-in-a-Box repository
  • Bulk Upload: Users upload large numbers of datasets simultaneously in a single zip file; metadata records are generated automatically.

To proceed, click either Individual Upload or Bulk Upload; these options are described below.

Individual Uploads

Performing an individual upload is a three-step process:

  • Create a Collection: Create a collection, in which you will store your metadata record(s)
  • Add Data: Add one or more metadata records to the collection; each metadata record describes a data resource, which will be either specified or uploaded in this step
  • Collection Metadata: Provide metadata that applies to all metadata records in your collection

These steps are described further in the following sections

Individual Upload Step 1: Create a Collection

Clicking Individual Upload on the Contribute page will bring you to the Create a Collection form; as the name implies, populating this form will create a collection, which is the first step of the Individual Upload process. The fields on this form are as follows:

  • Title: The title of your collection
  • URL: The URL at which your collection can be found. This URL is automatically generated when you enter a Title, but you can modify it manually if you wish
  • Description: A blurb providing a description of your collection and the metadata records stored in it
  • Tags: Keywords that can be used to search for your collection and any metadata records within it
  • License: The license under which any dataset(s) described by metadata records within this collection are published; among other things, the license determines the manner in which the dataset described by the metadata in this collection can be reused

When you are finished, click Next: Add Data.

Individual Upload Step 2: Add Data

Clicking Next: Add Data on the Create Collection form brings you to the Add Data. Here, you create a metadata record describing a specified data resource and add this metadata record to the collection you created in Step 1. Fields found on this form regardless of data resource type are as follows:

  • Resource: The URL by which the data resource described by your metadata record can be accessed; if you choose to upload a data resource to the NGDS Node-in-a-Box Repository, this field will be automatically populated
  • Name: The name of the data resource described by your metadata record as it will appear in the metadata catalog
  • Description: A brief description of the data resource described by your metadata record

Data resource types are as follows:

  • Structured Resource: Data that has been mapped to an interchange format such as an NGDS data exchange content model; if your data has been mapped to an NGDS data exchange content model, you will be given the opportunity to specify the content model to which your data has been mapped. Fields specific to this resource type are as follows:
    • Content Model: The NGDS content model to which the data resource described by your metadata record conforms
    • Content Model Version: The NGDS content model version to which the data resource described by your metadata record conforms
    • Distributor: The individual or agency responsible for distributing the data resource described by your metadata record
    • File Format: The file format of the data resource described by your metadata record
  • Unstructured Resource: Data such as plain text, a scanned image, or a sound file that cannot be processed or analyzed by machines without user input. Fields specific to this resource type are as follows:
    • Distributor: The individual or agency responsible for distributing this dataset described by your metadata record
    • Format: The file format of the data resource described by your metadata record
  • Offline Resource: A data resource that is only available in print format. Fields specific to this resource type are as follows:
    • Ordering Procedure: The manner in which the data resource described by your metadata record can be procured
  • Data Service: Data that is available as a web service; here, you will be asked to specify the type of web service. Fields specific to this resource type are as follows:
    • Distributor: The individual or agency responsible for distributing the data resource described by your metadata record
    • Protocol: The specific web service protocol used by the web service you have specified as the data resource described by your metadata record; examples include:
      • WMS: Open Geospatial Consortium (OGC) web map service (WMS); typically used to provide raster images of georeferenced features
      • WFS: Open Geospatial Consortium (OGC) web feature service (WFS); typically used to provide georeferenced features described by attributes
      • ESRI Map Service: Protocol used by web services generated through ESRI ArcGIS software; provides both georeferenced rasters and georeferenced features described by attributes
      • CSW: Open Geospatial Consortium (OGC) catalog service for the web (CSW); typically used to store and exchange metadata records; these metadata records are often used to locate other web services
      • OPeNDAP: Web service protocol typically used to provide georeferenced raster datasets
    • Layer: The label for a given data service when it is viewed as a layer in a GIS application

When you have populated the appropriate fields, you have two choices:

  • Click Save & add another to add another metadata record to the collection; this will repeat the steps described in Step 2
  • Click Next: Additional Info to finalize your metadata record by adding collection metadata

Individual Upload Step 3: Collection Metadata

Clicking Next: Additional Info brings you to the Collection Metadata form. Here, you will provide metadata that applies to the collection you created in Step 1 and the metadata records within it you created in Step 2.

The Collection Metadata form includes the following fields:

  • Visibility: Determines whether or not your collection and the metadata records within it are visible to the public
  • Authors: The authors of the data resource(s) described by metadata record(s) in the collection
  • Maintainer: The individual or agency responsible for the data resource(s) described by metadata record(s) in the collection
  • Spatial Location: A rough approximation of the features in the data resource(s) described by metadata record(s) in the collection
  • Category: A description of the type of data resource(s) described by metadata record(s) in the collection
  • URI: A uniform resource identifier for the collection; see the USGIN URI Tutorial for more information
  • Quality: A rough assessment of the quality of the data resource(s) described by metadata record(s) in the collection
  • Lineage: The provenance of data resource(s) described by metadata record(s) in the collection
  • Status: The workflow status of data resource(s) described by metadata record(s) in the collection
  • Publication Date: The date on which data resource(s) described by metadata record(s) in the collection were published
  • Language: The language in which the data resource(s) described by metadata record(s) in the collection are written
  • Dataset Extent: The physical extent of features in the data resource(s) described by metadata record(s) in the collection; click the Draw a rectangle button (image) to draw a box indicating the dataset extent

When you are done, click Finish. This will finalize your collection and all metadata records within it; these records will be added to the catalog.

Bulk Uploads

Clicking Bulk Upload brings you to the Bulk Upload form. Here, you have the opportunity to upload a .ZIP file containing numerous data resources; the Bulk Upload feature will automatically generate metadata records for each data resource in the ZIP file

To proceed, click the Choose File button on the second row; in the window that appears, locate the desired .ZIP file and click Open. In the Bulk Upload form, click Submit.
